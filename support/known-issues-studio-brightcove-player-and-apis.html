    Problemas conocidos: Studio y reproductor de Brightcove

    Entre los problemas conocidos identificados se incluyen:

    • Estudio

      • Vídeo en directo

    Studio

    El uso de la función Traducción de Chrome rompe algunas funciones de Studio
    El uso de la función de traducción de Chrome para traducir módulos de Studio hará que algunas funcionalidades se rompa.

    Módulo Media

    Captura de imagen con IE 11
    La captura de imágenes con IE 11 no funcionará cuando la relación de aspecto del vídeo es 1:1 o 4:3.
    Nombre del archivo de origen que contiene caracteres de doble byte
    El nombre del archivo de origen que contiene caracteres de doble byte se desconfía después de la retranscodificación en el módulo de medios.
    Compartir medios

    A la hora de compartir grandes cantidades de vídeos, Brightcove recomienda hacerlo como máximo en dos páginas de vídeo cada vez. Compartir muchos vídeos de forma simultánea puede provocar un error de tiempo de espera.

    No se admite la adición de subtítulos a un vídeo compartido que ya contenga subtítulos.

    El uso compartido de medios fallará en los vídeos creados al recortar transmisiones en vivo.

    Videos programados
    Dado que la API de reproducción y los vídeos en caché del catálogo durante un máximo de 10-15 minutos, es posible que un reproductor que solicite un vídeo programado para estar disponible durante los próximos minutos (hasta 20) no pueda obtener un vídeo reproducible hasta que se actualice la caché.

    Módulo Players

    Configuración de precarga
    Debido a un error en Internet Explorer, la configuración de precarga puede ignorarse al usar IE.
    Configuración Acceder a orígenes de datos entre dominios
    Si esta configuración de directiva de Internet Explorer está habilitada para un reproductor de Brightcove restringido por dominio, el reproductor no se cargará en Internet Explorer 11. Esto parece ser un error en IE, y no hay una solución alternativa conocida. Desactiva esta opción si quieres que los jugadores con dominio restringido funcionen en IE 11.

    Vídeo en directo

    El reproductor de Brightcove sigue contando visionados incluso cuando el evento en directo ha terminado
    El reproductor de Brightcove aumenta el contador video_view cada vez que se carga un activo de vídeo remoto, incluso aunque se trate de un evento en directo ya terminado con una lista de reproducción HLS vacía o ya no consultable. Solución alternativa: desactivar, anular la programación o eliminar el vídeo.
     

    Analytics

    El informe de ancho de banda total sólo está disponible para vídeos de ingesta heredados
    El campo de datos Ancho de banda total no está disponible actualmente para Entrega dinámica, por lo que esta opción para informes personalizados no estará disponible.
    Es posible que los datos de análisis no sean precisos debido a bloqueadores de anuncios
    Los datos de análisis (obtenidos tanto a través de la interfaz de usuario como de la API) pueden ser inexactos debido a que los bloqueadores de anuncios, como uBlock, bloquean el envío de datos analíticos en los equipos cliente.
    Informes personalizados Visualización de 0 bytes_entregado para algunos vídeos
    Los vídeos que se ingieren con un perfil de ingesta de entrega dinámica mostrarán 0 bytes_delivery en informes personalizados.
    Informes personalizados con varios filtros
    Se aplicará la operación AND cuando se utilicen múltiples (dos o más) filtros. Por ejemplo, si se añaden los dos filtros etiqueta de vídeo y reproductor, solo se obtendrán aquellos vídeos que contengan las etiquetas especificadas y que se reproduzcan en el reproductor especificado.
    Informes personalizados mediante la funcionalidad Filtrar por etiqueta de vídeo
    A la hora de crear informes personalizados, existe un límite para el uso de la función Filtrar por etiqueta de vídeo. El número de vídeos que pueden asociarse a una etiqueta específica en su biblioteca se limita a 2.000. Si el número de vídeos asociados a una etiqueta es mayor, recomendamos generar un informe no filtrado (es decir, que incluya todas las etiquetas) y que el filtrado se ejecute a nivel local en una hoja de cálculo.
    "Otro/Ajeno" aparece en el informe de rendimiento
    Si se reproduce en un reproductor de Video Cloud un vídeo que no se encuentra en su biblioteca de Video Cloud (por ejemplo, un activo remoto), en pantalla aparecerá Otro/Ajeno sin ningún identificador de referencia asociado.
    Los nombres actualizados de los vídeos no se reflejan en los análisis
    La única vez que el módulo Analytics "conoce" el título de un vídeo es cuando el reproductor se lo envía para comunicarle el comienzo de la reproducción. Si se cambia el título de un vídeo, el módulo Analytics no obtendrá el nuevo título hasta que el vídeo se reproduzca por primera vez. Por este motivo, si no se registra ninguna reproducción de un vídeo, puede ocurrir que en los informes del módulo Analytics siga figurando el nombre antiguo aunque el vídeo ya tenga uno nuevo. Los datos correspondientes a los periodos anteriores a marzo de 2013 a menudo tienen este problema, y mostrarán metadatos solo numéricos. Por ejemplo, un título de vídeo puede aparecer como 1230123012. Esto se debe a que los datos de este periodo fueron capturados en el sistema de análisis antiguo.
    En la interfaz de usuario del módulo Analytics aparecen los nombres de los vídeos y de los reproductores con ???????
    Los metadatos (nombres de vídeos y de reproductores) correspondientes al mes de octubre de 2013 pueden contener una serie de interrogaciones ?????. Este problema puede afectar a los editores que utilicen metadatos con caracteres de dos bytes y cuyos vídeos hayan sido visionados durante el mes de octubre de 2013. Este problema estaba relacionado con la forma en que capturábamos los metadatos para presentarlos en la interfaz de usuario del módulo Analytics. Desde noviembre de 2013, el módulo Analytics dispone de mecanismos adicionales para evitar la corrupción de los metadatos enviados con las métricas de los eventos.
    Zona horaria de los informes
    La zona horaria de generación de informes se utiliza para calcular los comienzos de los días reflejados en los informes. La modificación del ajuste de zona horaria para informes solo afecta a los datos desde el momento de efectuarse ese cambio, es decir, que no se aplica con carácter retroactivo. Si modifica este ajuste, puede que en los datos resultantes aparezca un tramo plano o un pico en los límites del día en que se efectuó esa modificación. Puede que la modificación de este ajuste no entre en vigor inmediatamente y que, por tanto, los datos de análisis sigan correspondiendo al ajuste anterior hasta que el sistema procese los cambios.
    Los datos de interés despertado se registran con una periodicidad diaria
    Los datos de interés despertado se registran con una periodicidad diaria, con las horas expresadas en UTC. En ocasiones, estos datos pueden incluir 2 días completos en la tabla "Audiencia al x%" en el informe de interés despertado.
    Soporte de Internet Explorer
    El módulo Analytics requiere Internet Explorer 11 o posterior.
    Alto número de resultados «otros» para el fabricante de dispositivos
    En algunos casos, es posible que vea un gran número de resultados «otros» para el fabricante del dispositivo. Esto ocurre cuando el recopilador de datos no reconoce el que se le ha user_agent entregado con datos de análisis. La causa más común que hemos identificado para esto es una cadena user_agent personalizada creada por Instagram, pero probablemente haya otras.

    Audience

    La actividad de la audiencia muestra Porcentaje observado:0 y Tiempo observado:0
    A veces, los eventos de interactividad se activan sin que se reproduzca un vídeo o antes de que un vídeo haya enviado su primer baliza. Esto puede dar lugar a que se graben eventos de visualización que tienen interactividad pero sin actividad de vídeo (0% visto, 0 segundos).
    Soporte de Internet Explorer
    Los formularios de clientes potenciales de audiencia requieren Internet Explorer 11 o posterior.

    Red social

    El plugin del navegador Ghostery puede interferir con el módulo Social

    El plugin del navegador Ghostery puede interferir con el módulo Social evitando que se cargue correctamente.

    Solución: Añade Studio a la lista blanca de Ghostery.

    Módulo Upload / Ingestión dinámica

    Las cargas a través del módulo de carga fallan si el reloj del dispositivo no coincide con la hora real
    Las cargas están autorizadas durante un cierto período de tiempo. Si la hora del reloj varía de la hora real en más de 15 minutos, recibirá errores al cargar.
    La retranscodificación mediante ingestión dinámica actualiza la fecha de activación del vídeo
    Al retranscodificar un vídeo mediante ingestión dinámica, la fecha de activación del vídeo se actualiza con la fecha actual. El uso de listas de reproducción inteligentes ordenadas por fecha de activación afecta al orden de aparición de los vídeos en la lista de reproducción.
    Nombres de archivo
    Los nombres de archivos de vídeo (incluida la extensión) no deben superar los 120 caracteres de un solo byte (60 bytes de doble byte). Si lo hace, el video se ingerirá con éxito, pero no podrá retranscodificarlo más tarde.

    Reproductor Brightcove Player

    Consulte las Notas de lanzamiento de Brightcove Player para saber si los problemas que presentaba el reproductor en el pasado se han corregido en una versión reciente.

    Elementos con id Establecer en default
    Brightcove Player es conocido por comportarse mal si hay un elemento con id="default" en la página. La entrada del blog DOM: los ID de elemento son variables globales explica por qué se trata de un problema.
    Anuncios bloqueados en Safari y/o iOS
    Los cambios realizados por Apple en las políticas de origen cruzado para Safari e iOS pueden provocar que los anuncios no se reproduzcan.

    Solución

    Hemos resuelto este problema para SSAI agregando encabezados CORs adicionales. Si utiliza anuncios del lado del cliente, esto no solucionará el problema, ya que los encabezados CORs deben enviarse desde su servidor o servicio de alojamiento. Los encabezados que hemos añadido son:

    access-control-allow-headers: Server,Range,Content-Length,Content-Range
          access-control-allow-headers: X-Requested-With,Origin,Range,Accept-Encoding,Referer
          access-control-allow-methods: GET,HEAD,OPTIONS
          access-control-allow-origin: *

    Vídeos sin póster o imágenes en miniatura
    Los vídeos sin miniaturas o imágenes de póster pueden provocar un error en el reproductor cuando se carga. Esto afecta a los reproductores de SDK móviles de una manera que puede bloquearlo.
    Internet Explorer siempre restablece su velocidad de reproducción a 1.0 siempre que se pausa la reproducción.
    Esto ya no será un problema en Safari 14 (versión Big Sur).
    Imágenes IE11 para contenido solo de audio que no se muestra
    Cuando se reproduce solo contenido de audio en IE11, la imagen fija de vídeo no se mostrará, solo se verá una pantalla negra. La imagen se puede conservar con el siguiente código: 
      .vjs-has-started .vjs-poster {
  display: block;

    Auto-Avance de la lista de reproducción en Safari
    En Safari, si la lista de reproducción está visible y el reproductor tiene la configuración de precarga establecida en ninguno, la reproducción automática del siguiente vídeo fallará silenciosamente.
    Menú contextual
    Al hacer clic con el botón derecho en un vídeo cerca de los bordes derecho o inferior de un reproductor, el menú contextual del reproductor se muestra fuera de la pantalla.
    Usar varios jugadores de diferentes tamaños con el mismo player_id en una página

    Si varios jugadores tienen el mismo id en una página y especifican diferentes tamaños dentro de la <video> etiqueta usando width y height atributos, se aplica el CSS del reproductor del último jugador a todos los jugadores.

    Soluciones provisionales

    • Utiliza jugadores con diferentes identificadores de jugador si quieres que tengan diferentes tamaños o estilos/li>
    • Defina clases para los reproductores que proporcionen el tamaño del reproductor mediante CSS, y agregue las clases a las etiquetas de vídeo para los diferentes reproductores
    FastClick.js provoca un manejo de eventos incorrecto
    FastClick.js provoca un manejo de eventos incorrecto en nuestra barra de control personalizada, lo que puede imposibilitar el uso de dicha barra en dispositivos móviles.
    Los vídeos muestran un tono grisáceo en Chrome y Firefox

    En ocasiones, los vídeos del reproductor de Brightcove ejecutados en Chrome y Firefox muestran un tono grisáceo. Esto puede deberse a la aceleración de hardware o a la configuración de los controladores de NVIDIA.

    Solución: Abra el panel de control de NVIDIA. En Vídeo, seleccione Ajustes de color del vídeo. En Cómo realizar los ajustes de color, seleccione Con los ajustes de NVIDIA. En Avanzado, asegúrese de que el Rango dinámico sea Completo (0-255) y no Limitado (16-235). La Mejora de contraste dinámica debe estar desactivada. Este problema se ha mencionado en el Foro de ayuda de Google Chrome.

    Uso de emuladores

    Si utiliza el modo dispositivo de Chrome u otros emuladores para la reproducción de sus vídeos de prueba, ha de tener en cuenta que los emuladores no representan de forma fidedigna el rendimiento real de un dispositivo. Aunque se pueden usar para realizar pruebas iniciales durante el desarrollo, lo más indicado para obtener resultados precisos es emplear dispositivos reales.

    Al poner a prueba la reproducción en el modo dispositivo de Chrome, puede aparecer el siguiente mensaje: "El uso de Chrome en modo dispositivo solo ofrece una representación del viewport y la cadena del agente de usuario de dicho dispositivo en Chrome, por lo que no representa de manera precisa el rendimiento real del dispositivo".

    Uso de data-setup
    Debe evitar el uso de data-setup con el reproductor de Brightcove. En ocasiones, el uso de data-setup aparece recogido en la documentación de la API, pero esto se debe a que la documentación se genera directamente a partir del código fuente Video.js del reproductor, y ese atributo se PUEDE utilizar con la etiqueta video en Video.js puro. El atributo envía información de la confirmación al reproductor, pero el reproductor de Brightcove emplea un método diferente para hacerlo, por lo que data-setup resulta poco fiable.
    Selección de una fuente con reconocimiento de protocolo y DASH
    La selección de una fuente con reconocimiento de protocolo no está disponible cuando se utiliza contenido DASH. Solo se puede utilizar con contenidos HLS y MP4.
    Error en la consola cuando se utiliza la implementación de reproductores iframe en Safari
    Cuando se utiliza la implementación de iframe del reproductor de Brightcove en Safari, aparece el siguiente mensaje de error en la consola: Se ha impedido que un fotograma con origen "http://players.brightcove.net" accediese a un fotograma con origen "#DOMAIN NAME#". Los protocolos, los dominios y los puertos deben coincidir. Esta notificación no afecta a la reproducción.
    En el menú de subtítulos de Safari aparecerá el mensaje "Desconocido"
    Esta es una limitación de Safari/Apple. Las últimas versiones del reproductor de Brightcove utilizan una función de subtitulación nativa integrada en Safari, y este es el comportamiento estándar. Apple lo documenta aquí: https://developer.apple.com/library/ios/qa/qa1801/_index.html
    Pantalla completa en entornos no de Flash

    En entornos en los que Flash o HLS basado en Flash esté desactivado para la reproducción y el reproductor tenga explícitamente asignado un tamaño por medio del atributo style de la etiqueta video , en algunos navegadores puede no funcionar la visualización a pantalla completa.

    Solución alternativa: elimine el atributo de la etiqueta y, en su lugar, cree una regla en la hoja de estilo de página como esta:style video 

      .video-js{
  width:640px;
  height:360px;
  }
    Compartición en redes sociales
    En equipos de escritorio, el botón para compartir en redes sociales no será visible durante la reproducción de anuncios.
    HTTPS

    En la mayoría de navegadores de escritorio, el reproductor de Brightcove solo reproducirá HLS en sitios web HTTPS cuando tanto el manifiesto como los segmentos de vídeo se transmitan a través de conexiones HTTPS. Esto se debe a las recientes modificaciones incorporadas a diversos navegadores, que restringen más severamente los contenidos no protegidos mediante SSL. El problema afecta a los usuarios de Chrome, Firefox e Internet Explorer en ordenadores de escritorio. No afecta a los usuarios de Safari ni a los navegadores de dispositivos móviles, como tampoco a la reproducción de variantes de representación MP4.

    Estamos trabajando en corregir esta limitación para recursos gestionados a través de Video Cloud. Si usted gestiona su propia CDN y su transcodificación (activos remotos), deberá configurar su CDN para que soporte entregas a través de HTTPS tanto del manifiesto como del contenido en vídeo.

    Visualización a pantalla completa

    En los navegadores más recientes compatibles con la API de pantalla completa, es preciso aplicar reglas CSS incrustadas en páginas para asegurarse de que el reproductor se escale al 100 % al activar la pantalla completa. De lo contrario, el reproductor aparecerá con el tamaño original dentro de la visualización a pantalla completa. Para obtener más información, consulte el artículo Visualización a pantalla completa del documento Dimensionar el reproductor.

    En el caso de IE10 y versiones anteriores, incompatibles con la API de pantalla completa, se abrirá una nueva ventana, pero el reproductor no se redimensionará para ocupar toda la ventana. Esto se debe a que no se han aplicado estilos para cambiar el tamaño del reproductor. Debido a que la imagen no se puede desplazar dentro de la ventana, puede que solo se vea una parte de la página web y que en ella no aparezca ningún reproductor en absoluto.

    Varios vídeos se publican en una sola página con capítulos de HapyAK

    Cuando se publican varios vídeos en una sola página con capítulos de HapyAK, podría aparecer un capítulo incorrecto.

    Solución

    Utilice esta secuencia de comandos: https://github.com/brightcove/gallery-in-page-snippets/blob/master/snippets/hapyakFix.js. Sin embargo, la reproducción Flash (IE11/Win7) no puede evitar este problema, por lo que la reserva HTML5 debe implementarse en caso de IE11/Win7.

    Akamai HD y HDS
    Brightcove Player no permite entregar vídeo con los métodos HD ni HDS de Akamai.
    Error: d.off no es una función

    Este error, causado por un error conocido, puede producirse al cambiar en un reproductor entre diferentes formatos, por ejemplo entre MP4 y HLS. Mientras se soluciona el problema, simplemente intente volver a ejecutar el código que está causando el problema. El código siguiente es un ejemplo que corrige el error en una aplicación:

      try {
  myPlayer.catalog.load(video);
  } catch (e) {
  myPlayer.catalog.load(video);
  }
  myPlayer.play();
    Duración de los vídeos HLS
    La duración que aparece en la barra de control puede ser distinta de la que se muestra inicialmente. La duración puede actualizarse cuando se hayan cargado todos los segmentos en formato HLS.
    RequireJS
    Si utiliza RequireJS, PUEDE que tenga que utilizar el método bc() para generar instancias del reproductor. La necesidad de utilizar el método vendrá determinada por cómo y cuándo se carguen los activos del reproductor. Para obtener más información, consulte el documento RequireJS y el reproductor de Brightcove.
    referrer_url frente a description_url
    El valor puede tener diferentes valores entre dispositivos iOS y Android.referrer_url Por ello, se recomienda usar en su lugar el valor description_url , que se mantiene constante en todos los dispositivos y plataformas.
    atributo de bucle en Safari
    El atributo loop no funciona correctamente en Safari. Puede utilizar la solución detallada en el ejemplo del reproductor de BrightCove:  Creación de un documento de bucle de vídeo .
    Subtítulos en Safari
    Debido a que Safari utiliza las capacidades nativas para aplicar estilos a los subtítulos, la opción de menú Configuración de subtítulos no está disponible en el menú CC en los navegadores Safari (tanto iOS como macOS).
    Usar un identificador de elemento HTML denominado global
    No debe anidar su código de implementación de reproductor avanzado (incrustar en página) en una etiqueta principal con un valor asignado.<div> id global Esto provoca problemas con Brightcove Player.
    Problema del icono de la versión 5
    Brightcove Player versión 5 utiliza el mismo icono para capítulos y subtítulos. Dado que la versión 5 está en modo de mantenimiento, y esto es un problema cosmético, es dudoso que el problema se solucionará.

    Android

    • El control de progreso y el control deslizante de volumen de Brightcove Player no se pueden ajustar con los gestos nativos de TalkBack en Android. Esto se debe a las limitaciones de la plataforma. En su lugar, un usuario debe interactuar directamente con el elemento. En Android, un doble toque con dos dedos permitirá al usuario ajustar el progreso o el volumen a la posición del doble toque con dos dedos.
    • En los dispositivos, puede que la barra de progreso no esté sincronizada con vídeos HLS, ya que HLS no funciona muy bien en Android. También puede ocurrir que la duración total del vídeo se indique incorrectamente, como 0:01.
    • En todas las versiones de Android existen múltiples problemas de accesibilidad relacionados con el navegador nativo tanto de teléfonos como de tabletas. Talkback no confirma mediante vibración la utilización de ningún control del reproductor. (Este problema no afecta al complemento de navegador Chrome en dispositivos Android, sino solo al navegador nativo.
    • En los dispositivos, al tocar la pantalla para confirmar mensajes de error no se abre una burbuja hasta el elemento de vídeo predecesor. Esto impide cerrar mensajes de error cuando aparecen.
    • En dispositivos Android, al navegar por los menús del reproductor en la barra de control, como subtítulos o selección de calidad, el menú puede quedar «atascado» en el estado abierto después de una larga pulsación sobre un elemento del menú. Esto se debe a que Chrome agrega la pseudo-clase.:hover Otra pulsación larga en otra parte del reproductor normalmente cerrará el menú.

    Averigüe cómo crear aplicaciones de Android que utilicen el SDK de reproductores de Brightcove para Android.

    iOS

    • El control de progreso y el control deslizante de volumen de Brightcove Player no se pueden ajustar con los gestos nativos de VoiceOver en iOS. Esto se debe a las limitaciones de la plataforma. En su lugar, un usuario debe interactuar directamente con el elemento. En iOS, el usuario debe pulsar dos veces para seleccionar la barra de progreso o el control deslizante de volumen y, a continuación, pulsar dos veces y mantener pulsado permitirá al usuario ajustar la hora o el volumen.
    • Safari en iOS nunca se carga previamente. Para obtener más información, consulte la Guía de audio y vídeo HTML5 de Safari.
    • El error del reproductor se produce en iOS al cambiar a otra aplicación. Cuando se reproduce un vídeo en Brightcove Player en un dispositivo iOS en Safari/Chrome, el reproductor se desactivará después de cambiar a otra aplicación y, a continuación, volver a Safari/Chrome después de un minuto o más.

    iPhone

    • No se puede compartir en redes sociales desde teléfonos iPhone. Debido a que los teléfonos iOS cambian a pantalla completa nativa, no se pueden compartir vídeos desde teléfonos iPhone.
    • En teléfonos iPhone, al tocar la pantalla para confirmar mensajes de error no se abre una burbuja hasta el elemento de vídeo predecesor. Esto impide cerrar mensajes de error cuando aparecen.

    Averigüe cómo crear aplicaciones de iOS que utilicen el SDK de reproductores de Brightcove para iOS.

    Windows 8

    • No se pueden incluir anuncios acompañantes.
    • En tabletas con Windows 8, la función de posicionamiento en un vídeo no funciona durante la reproducción de HTML/MP4 estándar. Funciona correctamente con la tecnología HLS.
    • En tabletas con Windows 8, los anuncios normales funcionan, pero con los anuncios acompañantes el navegador se cuelga.
    • En tabletas con Windows 8 se admiten vídeos HLS, Flash, en directo y todos los tipos de incrustación.
    • En teléfonos con Windows 8 se puede reproducir MP4, pero no Flash ni HLS.
    • En teléfonos con Windows 8, la reproducción siempre se inicia a pantalla completa tanto si el tipo de incrustación es iframe como inline. Esto impide mostrar las sobreimpresiones una vez iniciada la reproducción.
    • No se pueden activar los subtítulos.

    Chrome

    • Problema con las representaciones que tienen una velocidad de bits de audio baja

      Debido al error en MSE en la implementación del navegador Chrome documentada aquí:

      https://bugs.chromium.org/p/chromium/issues/detail?id=534301

      La reproducción en ese navegador para la versión 5 y superior del reproductor BC fallará (mostrando MEDIA_ERR_DECODE) si el perfil de audio de la copia que se intenta es diferente de AAC-LC.

      Para evitar que esto suceda en el nuevo contenido ingerido, los clientes deben asegurarse de que

      • utilizar una velocidad de bits de audio igual o superior a 48 kbps
      • incluya la siguiente configuración en su perfil DI: "max_aac_profile": "aac-lc"

      Para evitar que esto suceda con contenidos existentes, dispone de estas opciones:

      • retranscodificar siguiendo las recomendaciones anteriores
    • Puede haber pendiente una solicitud de vídeo HTML5 que impide que el vídeo se cargue. Consulte el documento de Google sobre este asunto.

    Firefox

    • Navegador Firefox para Android: Oficialmente incompatible, pero se intentará corregir los errores que sean posibles.
    • Algunos usuarios han comunicado que la versión 42 de Firefox puede tener problemas de reproducción con el reproductor Brightcove Player. Este problema parece estar relacionado con el ajuste de Firefox Utilizar aceleración de hardware. Con esta opción activada, el reproductor de vídeo solo reproducirá el audio y solo mostrará una imagen fija. Este problema se puede resolver desactivando esta opción como sigue: Opciones -> Avanzado -> General -> (DESMARQUE) Usar aceleración de hardware cuando esté disponible
    • El Reproductor de Brightcove puede agotarse el tiempo de espera en Firefox si el primer fotograma del contenido de vídeo tiene una marca de tiempo de presentación inicial (pts) mayor que 0. La solución recomendada actualmente es volver a codificar su contenido. Tenga en cuenta que volver a codificar NO significa volver a transcodificar. El maestro original tendrá que volver a cargarse. Si esto no soluciona el problema, póngase en contacto con el servicio de atención al cliente para obtener más ayuda.

    Internet Explorer

    • Los subtítulos configurados para que se muestren automáticamente en el vídeo (comprobar la configuración «predeterminada» para un título en el módulo multimedia) no funcionan con IE11
    • Los vídeos con audio superior a 48 kHz fallarán durante la reproducción en Edge e IE11 en Windows 8 y Windows 10. Esta es una limitación de Microsoft. Consulte este documento de Microsoft para obtener más detalles.
    • Los anuncios en Flash IMA3 funcionan mejor en IE. Google IMA3 no es compatible con anuncios en Flash y HTML en IE, aunque durante nuestras pruebas hemos detectado que el SDK para Flash es mucho más estable en este momento.
