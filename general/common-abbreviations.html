A

API
interfaz de programación de aplicaciones: una interfaz informática que define las interacciones entre varios intermediarios de software. Define los tipos de llamadas o peticiones que se pueden realizar, cómo realizarlas, los formatos de datos que se deben utilizar, las convenciones a seguir, etc. Una API puede ser completamente personalizada, específica de un componente, o puede diseñarse según un estándar de la industria para garantizar la interoperabilidad.

C

CDN
Red de entrega de contenidos - Red distribuida geográficamente de servidores proxy y sus centros de datos. El objetivo es proporcionar alta disponibilidad y rendimiento mediante la distribución espacial del servicio en relación con los usuarios finales. Las CDN surgieron a fines de la década de 1990 como un medio para aliviar los cuellos de botella de rendimiento de Internet.
CMS
sistema de gestión de contenidos - software informático utilizado para gestionar la creación y modificación de contenidos digitales. Hay muchos CMS web, como Wordpress y Drupal, que se utilizan para gestionar el contenido de un sitio web, pero los CMS se pueden utilizar para gestionar cualquier tipo de contenido. Por ejemplo, la biblioteca multimedia de Video Cloud es un CMS para contenido multimedia y activos multimedia, y es por eso que la API principal que se utiliza para administrar ese contenido se llama API CMS.

D

PIZCA
Transmisión dinámica adaptable a través de HTTP - una técnica de transmisión de velocidad de bits adaptativa que permite la transmisión de contenido multimedia de alta calidad a través de Internet entregado desde servidores web HTTP convencionales. Similar a la solución HTTP Live Streaming (HLS) de Apple, MPEG-DASH funciona dividiendo el contenido en una secuencia de segmentos pequeños, que se publican a través de HTTP.
DRM
Gestión de derechos digitales - las herramientas o medidas de protección tecnológica son un conjunto de tecnologías de control de acceso para restringir el uso de hardware propietario y obras protegidas por derechos de autor. Las tecnologías DRM intentan controlar el uso, la modificación y la distribución de obras con derechos de autor (como software y contenido multimedia), así como los sistemas dentro de los dispositivos que hacen cumplir estas políticas.

H

HLS
Transmisión en directo HTTP - un protocolo de comunicaciones de transmisión de transmisión de velocidad de bits adaptable basado en HTTP desarrollado por Apple Inc. y lanzado en 2009. El soporte para el protocolo está muy extendido en reproductores de medios, navegadores web, dispositivos móviles y servidores de medios de transmisión.
HTTP
Protocolo de Transferencia de Hipertexto - un protocolo de capa de aplicación para sistemas de información hipermedia distribuidos y colaborativos. HTTP funciona como un protocolo de solicitud-respuesta en el modelo informático cliente-servidor. Un navegador web, por ejemplo, puede ser el cliente y una aplicación que se ejecuta en una computadora que aloja un sitio web puede ser el servidor. El cliente envía un mensaje de solicitud HTTP al servidor. El servidor, que proporciona recursos como archivos HTML y otro contenido, o realiza otras funciones en nombre del cliente, devuelve un mensaje de respuesta al cliente. La respuesta contiene información sobre el estado de finalización de la solicitud y también puede incluir contenido solicitado en el cuerpo del mensaje.

METRO

MP4
MPEG-4 Parte 14 - un formato de contenedor multimedia digital que se usa más comúnmente para almacenar video y audio, pero también se puede usar para almacenar otros datos como subtítulos e imágenes fijas. A diferencia de HLS y DASH, MP4 se entrega como un único archivo en lugar de segmentos, por lo que se utiliza a menudo en los casos en que desea que los espectadores puedan descargar el vídeo.

R

DESCANSAR
Transferencia de estado representacional - un estilo de arquitectura de software que define un conjunto de restricciones que se utilizarán para crear servicios web. Los servicios web que se ajustan al estilo arquitectónico REST, llamados servicios web RESTful, brindan interoperabilidad entre sistemas informáticos en Internet. Los servicios web RESTful permiten a los sistemas solicitantes acceder y manipular representaciones textuales de recursos web mediante el uso de un conjunto uniforme y predefinido de operaciones sin estado. Brightcove utiliza API RESTful para acceder a varias partes de la plataforma.
RTMP
Protocolo de mensajería en tiempo real - inicialmente un protocolo propietario desarrollado por Macromedia para transmitir audio, video y datos a través de Internet, entre un reproductor Flash y un servidor. Macromedia ahora es propiedad de Adobe, que ha lanzado una versión incompleta de la especificación del protocolo para uso público.
