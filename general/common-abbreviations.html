    Abreviaturas comunes

    En este tema se explican las abreviaturas comunes que se utilizan en la documentación de Brightcove. Muchas de las definiciones son tomadas o adaptadas de Wikipedia.

    Índice

    | -A- | -B- | -C- | -D- | -E- | -F- | -G- | -H- -I- | -L- | -M-| -N- | -O- | -P- | -R- | -S- | -T- | -U- | -V- | -W- |

    A

    API
    interfaz de programación de aplicaciones - una interfaz informática que define las interacciones entre múltiples intermediarios de software. Define los tipos de llamadas o solicitudes que se pueden realizar, cómo realizarlas, los formatos de datos que se deben utilizar, las convenciones a seguir, etc. Una API puede ser totalmente personalizada, específica de un componente, o puede diseñarse basándose en un estándar de la industria para garantizar la interoperabilidad.

    C

    CDN
    Red de entrega de contenido - red distribuida geográficamente de servidores proxy y sus centros de datos. El objetivo es proporcionar alta disponibilidad y performance distribuyendo el servicio espacialmente en relación con los usuarios finales. Las CDN surgieron a finales del decenio de 1990 como medio para aliviar los cuellos de botella en el desempeño de Internet.
    CMS
    sistema de gestión de contenidos - software informático utilizado para gestionar la creación y modificación de contenido digital. Hay muchos CMS web, como Wordpress y Drupal, que se utilizan para gestionar los contenidos de un sitio web, pero los CMS se pueden utilizar para gestionar cualquier tipo de contenido. Por ejemplo, la biblioteca multimedia de Video Cloud es un CMS para contenido multimedia y activos multimedia, por lo que la API principal utilizada para administrar ese contenido se denomina API de CMS.

    D

    PIZCA
    Transmisión adaptable dinámica a través de HTTP : una técnica de transmisión de velocidad de bits adaptable que permite la transmisión de contenido multimedia de alta calidad a través de Internet entregado desde servidores web HTTP convencionales. Similar a la solución HTTP Live Streaming (HLS) de Apple, MPEG-DASH funciona dividiendo el contenido en una secuencia de pequeños segmentos, que se sirven a través de HTTP.
    DRM
    Gestión de derechos digitales - herramientas o medidas de protección tecnológica son un conjunto de tecnologías de control de acceso para restringir el uso de hardware propietario y obras protegidas por derechos de autor. Las tecnologías DRM tratan de controlar el uso, modificación y distribución de obras protegidas por derechos de autor (como software y contenido multimedia), así como de sistemas dentro de dispositivos que aplican estas políticas.

    H

    HLS
    HTTP Live Streaming - un protocolo de comunicaciones de transmisión de velocidad de bits adaptable basado en HTTP desarrollado por Apple Inc. y lanzado en 2009. El soporte para el protocolo está muy extendido en reproductores multimedia, navegadores web, dispositivos móviles y servidores multimedia de transmisión.
    HTTP
    Protocolo detransferencia de hipertexto: un protocolo de capa de aplicación para sistemas de información distribuidos, colaborativos y de hipermedios. HTTP funciona como protocolo de solicitud y respuesta en el modelo de computación cliente-servidor. Un navegador web, por ejemplo, puede ser el cliente y una aplicación que se ejecuta en un equipo que aloja un sitio web puede ser el servidor. El cliente envía un mensaje de solicitud HTTP al servidor. El servidor, que proporciona recursos como archivos HTML y otro contenido, o realiza otras funciones en nombre del cliente, devuelve un mensaje de respuesta al cliente. La respuesta contiene información sobre el estado de finalización de la solicitud y también puede contener contenido solicitado en el cuerpo del mensaje.

    M

    MP4
    MPEG-4 Parte 14 - un formato de contenedor multimedia digital más comúnmente utilizado para almacenar vídeo y audio, pero también se puede utilizar para almacenar otros datos como subtítulos e imágenes fijas. A diferencia de HLS y DASH , MP4 se entrega como un solo archivo en lugar de segmentos, por lo que a menudo se utiliza en los casos en los que desea que los espectadores puedan descargar el vídeo.

    R

    REST
    Transferencia de estado representacional : estilo arquitectónico de software que define un conjunto de restricciones que se utilizarán para crear servicios Web. Los servicios Web que se ajustan al estilo arquitectónico REST, denominados servicios Web RESTful, proporcionan interoperabilidad entre sistemas informáticos en Internet. Los servicios Web RESTful permiten a los sistemas solicitantes acceder y manipular representaciones textuales de recursos Web mediante un conjunto uniforme y predefinido de operaciones sin estado. Brightcove utiliza API RESTful para acceder a varias partes de la plataforma.
    RTMP
    Protocolo de mensajería en tiempo real - inicialmente un protocolo propietario desarrollado por Macromedia para transmitir audio, vídeo y datos a través de Internet, entre un reproductor Flash y un servidor. Macromedia ahora es propiedad de Adobe, que ha publicado una versión incompleta de la especificación del protocolo para uso público.
