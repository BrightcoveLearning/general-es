Vídeo MPEG-DASH

En este tema, aprenderá a crear representaciones MPEG-DASH para sus videos.

Descripción general

MPEG-DASH es un formato de video segmentado. Es particularmente útil si necesita seguridad DRM para su contenido, porque DASH se puede usar con Playready, Marlin o Widevine Modular DRM. Dado que la sintaxis para especificar las representaciones MPEG-DASH es ligeramente diferente a la de otros formatos de video, incluimos aquí un par de perfiles de muestra, uno sin empaquetado DRM y el otro con empaquetado DRM.

Para obtener detalles sobre cómo Brightcove Player determina qué interpretación reproducir realmente, consulte la Determinar qué interpretación se reproducirá documento.

Perfil de muestra (sin DRM)

Este ejemplo muestra el empaquetado de representaciones MPEG-DASH sin empaquetado DRM.

Para las representaciones DASH, establezca el decoder_bitrate_cap decoder_buffer_size para optimizar la reproducción en los entornos de visualización de destino.

Nuestra recomendación general es establecer decoder_bitrate_cap a la tasa de bits máxima disponible para la transmisión de video. Recuerde que debe tener en cuenta el audio, así como la sobrecarga para el formato de archivo. La decoder_buffer_size se supone técnicamente que coincide con la memoria de búfer de vídeo disponible en el dispositivo de decodificación, pero a menudo se utiliza mejor para controlar la cantidad de contenido que se puede almacenar previamente en búfer. Funciona como múltiplo de los decoder_bitrate_cap, por ejemplo, si decoder_bitrate_cap era de 500 kbps, y decoder_buffer_size era de 1000 kbps, entonces el búfer podía almacenar vídeo por valor de 2 segundos. También recomendamos configurar el video_bitrate un poco más bajo que el decoder_bitrate_cap, lo que permite al codificador utilizar más datos para codificar escenas complejas y menos datos en escenas más sencillas. Un punto de partida decente sería asumir hasta un 10% de sobrecarga para el formato de archivo, estableciendo el video_bitrate 10% más bajo que el decoder_bitrate_cap y permitir que se almacenen en búfer aproximadamente 1,5 segundos de vídeo.

Suponiendo que hay una conexión de 1000 Kbps disponible para el espectador:

  • 1000 x 0,90 = 900 Kbps disponibles tras restar un 10% para la sobrecarga de formato.
  • 900-128 = 772 Kbps decoder_bitrate_cap de video después de restar 128 Kbps para el audio.
  • 772 * 1,5 = 1158 Kb decoder_buffer_size (durante 1,5 segundos de vídeo).
  • 772 * 0,90 = velocidad de bits de vídeo objetivo de 695 Kbps (10% menor que el decoder_bitrate_cap).
  1. El reproductor Brightcove actualmente admite la reproducción DASH solo con complementos especiales.
  2. La force_aac_profile La propiedad debe estar presente en cada versión DASH.
    • force_aac_profile debe uno de estos tres valores:
      • aac-lc
      • he-aac
      • he-aac-v2
    • El valor de force_aac_profile debe ser el mismo para todas las representaciones en un perfil

Seguridad Silverlight

En determinadas plataformas, Brightcove Player utiliza un respaldo de Silverlight para reproducir versiones MPEG-DASH. Por razones de seguridad, Silverlight requiere que un clientaccesspolicy.xml el archivo debe estar presente en la raíz del documento del servidor desde el que se sirve el contenido DASH. Ver este artículo de Microsoft para más detalles.

Problemas conocidos

  • La Predeterminado (visualización automática) ajuste de subtítulos en el Medios de comunicación módulo PISTA DE TEXTO La sección no es compatible cuando se usa junto con DASH y subtítulos en el manifiesto. Los subtítulos en el manifiesto se utilizan con los productos de Brightcove Dynamic Delivery y SSAI, por ejemplo. Una solución para este problema es utilizar el <track> etiqueta con la implementación de Advanced Brightcove Player. Esto se detalla en el Agregar subtítulos a videos mediante programación documento. Tenga en cuenta que debe utilizar el default atributo con el <track> etiqueta.
