Introducción

Las reglas de entrega de Brightcove son una función de Dynamic Delivery que le permite aprovechar la capacidad de generación de manifiestos justo a tiempo de manera más granular. Con las Reglas de entrega, puede crear reglas personalizadas para controlar cómo se entrega su contenido al espectador. Esto le permite entregar contenido de manera más eficiente y efectiva.

Las reglas de entrega le brindan la capacidad de crear lógica condicional basada en el tipo de dispositivo, la ubicación del usuario o una señal de su aplicación. Cuando se activan estas condiciones, usted controla el comportamiento, como los tipos de representaciones devueltas y qué CDN usar.

Ejemplos de

Como puede imaginar, las combinaciones de lógica condicional pueden ser infinitas. A continuación, se muestran algunos ejemplos de lo que pueden hacer las reglas de entrega:

Múltiples niveles de suscripción

Un modelo común para los servicios de suscripción de video a pedido (SVOD) es ofrecer dos niveles de suscripción: definición estándar (SD) y alta definición (HD).

Según una señal de su aplicación, puede decirle a Dynamic Delivery que devuelva solo representaciones SD para usuarios estándar y representaciones HD para usuarios premium.

CDN optimizados para geografía

Si bien la mayoría de las CDN están optimizadas para la distribución mundial, hay algunas áreas que están mejor atendidas por una CDN específica de la región. Se puede utilizar una CDN en la región o en el país para una entrega óptima.

Con las Reglas de entrega, puede crear una regla para que cuando un visor se encuentre en un área específica, se use una CDN específica de la región. Para todos los demás usuarios, se utilizará una CDN global.

Reproducciones optimizadas para dispositivos

Dado que los consumidores ven los medios desde una variedad de tamaños de pantalla, es menos que ideal entregar el mismo conjunto de representaciones en cada dispositivo.

Con las Reglas de entrega, puedes crear una regla que detecte si el dispositivo del espectador es un teléfono y excluya las copias de 1080p. Podría tener otra regla que detecte si un espectador está en un dispositivo OTT y excluir las representaciones por debajo de 360p.
