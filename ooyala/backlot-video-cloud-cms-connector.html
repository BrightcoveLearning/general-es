--- title: 'Conector CMS Backlot / Video Cloud' description: '

El Backlot API de CMS El adaptador proporciona soporte para un conjunto selecto de API de backlot operaciones para ayudarlo a mantener en funcionamiento las integraciones y las aplicaciones mientras realiza la transición a las API de Video Cloud. Para obtener más información sobre las API de la plataforma Video Cloud, consulte Introducción a las API de Brightcove.

Nota: este tema solo se aplica a los clientes de Ooyala que se están migrando a Video Cloud.

Requisitos

Tenga en cuenta que para que el adaptador de reproducción funcione, deberá trabajar con su CXM para realizar ajustes en la configuración de DNS en sus computadoras y dispositivos. Consulte Agregar el servidor DNS de Brightcove para obtener más información.

Extremos del conjunto de anuncios

Video Cloud no tiene el concepto de un ad_set, pero los datos de los conjuntos de anuncios asociados a un vídeo se migrarán a cue_points y ad_set_id se almacenará en un campo personalizado de ese nombre para permitirnos identificar vídeos asociados a un conjunto de anuncios.

Ver Migración a Video Cloud: Conjuntos de anuncios para obtener más información sobre cómo se asignan los datos del conjunto de anuncios a puntos de referencia para diferentes fuentes de anuncios.

Etiquetas de puntos finales

Las etiquetas se migrarán a Video Cloud como tags. A diferencia de las etiquetas, las etiquetas se establecen en videos individuales; no hay un "conjunto de etiquetas" a nivel de cuenta y las etiquetas no son jerárquicas. Las jerarquías de etiquetas se conservarán en etiquetas de Video Cloud como esta:

Aparte de esas diferencias, las etiquetas se pueden usar de muchas de las formas en que las etiquetas son, por ejemplo:

  • buscar videos que tengan algunas, todas o ninguna de un conjunto de etiquetas
  • Seleccionar videos para una lista de reproducción "inteligente" en tiempo de ejecución
Tenga en cuenta que si bien todos los /labels Los terminales son compatibles, algunas operaciones, en particular para las cuentas que tienen una gran cantidad de videos, llevarán más tiempo.

Puntos finales de reglas de publicación

Las restricciones de reproducción implementadas a través de reglas de publicación se aplican a activos individuales en Video Cloud. Las restricciones de dominio no se migran, porque en Video Cloud se establecen en reproductores, no en videos.

En activos de video
  • restricciones geográficas
  • programación (fecha y hora de inicio y finalización)

Las propiedades de las reglas de publicación de Ooyala relacionadas con las restricciones geográficas y la programación se aplicarán a los videos asociados con la regla de publicación, y id se almacenará en un campo personalizado con el nombre publishing_rule_id. los API de backlot Las operaciones de la regla de publicación funcionarán en todos los videos asociados con la regla de publicación, pero solo se devolverán las restricciones geográficas o de programación.
