Ooyala-Video Cloud: Ingesta de activos remotos a través de MRSS

Puede seguir utilizando MRSS para crear y actualizar activos remotos en Video Cloud.
Nota: estos temas se aplican solamente a los clientes de Ooyala migrados a Video Cloud.

Introducción

Puede agregar o actualizar activos remotos usando Media RSS (MRSS).

Ver Trabajar con activos remotos para una mejor comprensión de los activos remotos en Video Cloud.

Cuando usa MRSS para agregar o actualizar un activo remoto, identifica el activo como remoto especificando el < ooyala: activo remoto> etiqueta (en lugar de < Contenido multimedia> ) en el elemento de feed MRSS.

Nota: Para decidir si agregar o actualizar un activo remoto, Brightcove usa el <ooyala:embedcode></ooyala:embedcode> campo en el Manifiesto XML para determinar si el activo remoto existe actualmente (actualizar) o no (agregar).

Agregar un activo remoto

El siguiente código de ejemplo crea un activo remoto (el <guid> especificado no existe).

<rss xmlns:media=» http://search.yahoo.com/mrss/ "xmlns:dcterms=» http://purl.org/dc/terms/ "xmlns:fh=» http://purl.org/syndication/history/1.0 "xmlns:ooyala=» http://www.ooyala.com/mrss/ "version="2.0"> <channel> <item> <ooyala:duración de activos remotos en MS='1000' url=» http://mysite.com/RCTTestAssets/mrss/assets/Test.mp4"/ > <media:thumbnail url=» http://mysite.com/test.jpg"/ > <Media:subtitle href=» http://mysite.com/test.dfxp"/ > <Media:título>Prueba</media:title > <Media:Description> Prueba</media:description > <OOYALA:etiquetas>/Test</ooyala:labels > <link> http://mysite.com/videos/sports/test_0010.html</link > <! — establecer tiempos de vuelo —> <dcterms:valid> start=2011-06-28T 15:15:00-07:00; end=2022-12-01T 16:00:00-07:00; scheme=w3c-dtf </dcterms:valid > </item > </channel > </rss >

Actualización de un activo remoto

Para un activo remoto existente, puede actualizar los siguientes campos a través de MRSS:

  • medios: título
  • medios: descripción
  • ooyala: subtítulo
  • ooyala: metadatos
  • dcterms: válido
  • media: miniatura
  • ooyala: etiquetas

Cuando envía el elemento de feed de MRSS, si contiene actualizaciones de alguno de estos campos en el <item> etiqueta, el sistema actualiza el activo remoto con los cambios aplicables.
