Ooyala a la migración de Video Cloud: Adición del servidor DNS de Brightcove para pruebas

Para probar el Adaptador de reproducción y el Conector CMS que le permiten continuar usando reproductores Ooyala para entregar contenido de Video Cloud, deberá agregar un servidor DNS a sus sistemas y proporcionar información a Brightcove para la configuración. Este tema le guía a través de los pasos.

Nota: este tema se aplica solamente a los clientes de Ooyala que se están migrando a Video Cloud.

Introducción

Para prueba los Adaptador de reproducción que permitirá a sus reproductores de Ooyala entregar contenido desde Video Cloud y / o utilizar el Conector CMS que le permite utilizar algunas operaciones del API de backlot para administrar sus medios en Video Cloud, deben suceder dos cosas:

  1. Primero: comuníquese con su CXM para iniciar el proceso. Su CXM necesitará obtener sus direcciones IP y lo ayudará a encontrarlas si no sabe cómo hacerlo. Su CXM también puede ayudarlo con los pasos explicados en el resto de este documento que debe realizar.
  2. Lo que debe hacer para cada computadora y dispositivo que utilizará el adaptador de reproducción o el conector CMS es agregar un servidor DNS de Brightcove que redirigirá las solicitudes enviadas a los recursos de ooyala.com a las API de Brightcove correspondientes. Los pasos para hacer esto se detallan en las secciones siguientes.
Notas:
  • Tenga en cuenta que si realiza el cambio de DNS en una computadora o dispositivo antes de su dirección IP está incluida en la lista blanca de su CXM, puede deshabilitar el acceso a Internet.
  • Se requiere el servidor DNS adicional solo para probar. Una vez que haya verificado que puede usar los reproductores y servicios de Ooyala que necesita con los datos de Video Cloud, puede eliminar el servidor DNS de Brightcove de las computadoras y dispositivos que usa para probar.

Pasos de macOS

Suponiendo que no tiene una VPN, a continuación se muestran los pasos para agregar el servidor DNS de Brightcove en MacOS para computadoras portátiles y de escritorio Mac.

  1. Preferencias del sistema abierto:
  2. Abra las Preferencias de red:
  3. Clickea en el Avanzado ajustes:
  4. Abra la configuración de DNS:
  5. Haga clic en el + firmar para agregar un servidor DNS:
  6. Ingrese la siguiente dirección para el nuevo servidor DNS:

    35.170.73.174

  7. Haga clic en el + inicie sesión en Search Domains para agregar un dominio de búsqueda:
  8. Ingrese el siguiente dominio de búsqueda:

    ooyala.com

  9. Hacer clic OK para cerrar la configuración avanzada.
  10. Hacer clic Solicitar para guardar la nueva configuración.
  11. Reinicie su Mac para borrar la caché de DNS.

Pasos de Windows 10

A continuación se muestran los pasos para agregar el servidor DNS de Brightcove en Windows 10 para computadoras portátiles y de escritorio.

  1. Abra el Panel de control.
  2. Haga clic en Red e Internet.
  3. Haga clic en Centro de redes y recursos compartidos.
  4. Haga clic en la opción Cambiar la configuración del adaptador en el panel izquierdo.
  5. Haga clic con el botón derecho en la interfaz de red conectada a Internet y seleccione la opción Propiedades:
  6. Seleccione y active la opción Protocolo de Internet versión 4 (TCP/IPv4).
  7. Haga clic en el botón Propiedades.
  8. Haga clic en la opción Usar las siguientes direcciones de servidor DNS.
  9. Para su dirección de servidor DNS preferida en alternativa, ingrese 35.170.73.174
  10. Hacer clic OK.
  11. Hacer clic Cerrar
  12. Reinicie la computadora para borrar la caché de DNS.

Tenga en cuenta que el dispositivo debe tener una conexión Wi-Fi para realizar este cambio.

  2. Abra la configuración de Wi-Fi:

  4. Toque Configurar DNS para cambiar la configuración:

Caso especial: VPN

Es posible que los pasos anteriores no funcionen si su computadora / dispositivo está en VPN, porque el servidor VPN puede anular la configuración del servidor DNS con la suya propia. Si esto sucede, deberá agregar el servidor DNS de Brightcove a su VPN. Los comandos que necesitará exactamente dependerán de la plataforma y la VPN, pero a continuación se muestra una muestra para MacOS para darle una idea.

Obtenga la subclave para la entrada DNS en la interfaz VPN. Por ejemplo, con Pulse Secure, se ve así:

sudo scutil
> get subKey [127 = State:/Network/Service/net.pulsesecure.pulse.nc.main/DNS]
> d.add ServerAddresses * 35.170.73.174
> set State:/Network/Service/net.pulsesecure.pulse.nc.main/DNS
