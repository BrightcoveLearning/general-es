Nota: este tema se aplica solamente a los clientes de Ooyala migrados a Video Cloud.

Nuevas opciones de Video Cloud

Video Cloud tiene tres opciones estándar para la ingesta de videos:

  • El módulo de carga en Studio
  • Ingestión basada en extracción mediante la API de ingesta dinámica
  • Carga del archivo de origen mediante la API de ingesta dinámica

O use los métodos de ingestión de Ooyala

Como cliente de Video Cloud, puede utilizar cualquiera de las opciones enumeradas anteriormente, pero Brightcove también ha creado adaptadores para los sistemas de ingesta de Ooyala para dirigir los videos al sistema de ingesta de Video Cloud, de modo que pueda continuar utilizando el Servicio de ingestión de Ooyala (OIS) para enviar sus videos a Video Cloud usando:

  • FTPS
  • Aspera
  • Alojar feeds MRSS de forma remota
  • La API de Backlot

Todas las opciones de manifiesto existentes seguirán funcionando. Continúe usando sus credenciales de autenticación para Ooyala, y el adaptador las traducirá automáticamente en credenciales para las API de Video Cloud necesarias:

Ingestión basada en API
Si su flujo de trabajo de ingestión incluye la ingesta de contenido nuevo directamente a través de la API de ingestión: cuando el tráfico de su cliente se dirija a Video Cloud, utilizará la clave y el secreto de la API de Ooyala existente para realizar las llamadas de la API a los puntos finales de la API de Ooyala actuales.
Ingestión basada en FTP / Aspera
Asegúrese de restablecer la contraseña de la cuenta de usuario "Sólo se pudo cargar" de video a algo diferente a lo que tenía en Backlot. Si su cuenta de "solo carga" en Backlot admitía tanto video como metadatos, entonces la cuenta de "solo carga" de Video Cloud también lo hará. Puede continuar conectándose a los mismos puntos finales FTP / Aspera y usar sus nuevas credenciales de cuenta de usuario de "solo carga" de Video Cloud para cargar archivos. Una vez que los archivos se hayan cargado y hayan desaparecido de la carpeta de carga, inicie sesión en Video Cloud y asegúrese de que se haya agregado su video. Si está cargando archivos de metadatos XML o CSV junto con el video y desea apuntar a un perfil de ingestión específico, deberá especificar el nombre del perfil de Video Cloud en el campo XML o CSV en el elemento profileguid. Ver Gestión de perfiles de ingesta.
Ingestión basada en MRSS
Comparta su feed de MRSS con su equipo de cuentas y ellos podrán habilitarlo. Debido a que no podremos validar qué videos del feed de MRSS se ingirieron previamente en Backlot, asegúrese de que solo los videos nuevos estén disponibles en el feed de MRSS. Si hay videos antiguos disponibles en el feed, también se incorporarán, lo que dará como resultado contenido duplicado en Video Cloud. Una vez que la fuente de MRSS está en su lugar, puede iniciar sesión en Video Cloud para validar que el contenido se está ingiriendo.
Ingestión a través de Brightcove Video Cloud Studio
Una diferencia importante con Backlot es el perfil de ingestión que seleccionará para ingerir su contenido; asegúrese de elegir un perfil de ingesta en Video Cloud que mejor se adapte a sus requisitos, puede encontrar información útil sobre los perfiles de ingesta de Video Cloud en Gestión de perfiles de ingesta.
