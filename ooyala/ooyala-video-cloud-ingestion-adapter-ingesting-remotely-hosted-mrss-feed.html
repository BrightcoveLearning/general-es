Ooyala-Video Cloud: Ingesta desde un feed MRSS alojado de forma remota

Puede continuar ingiriendo videos y activos en Video Cloud usando feeds MRSS a través del Adaptador de ingestión.
Nota: este tema se aplica solamente a los clientes de Ooyala migrados a Video Cloud.

Introducción

Puede usar Media RSS (MRSS) para que Ooyala extraiga su contenido de su propio sistema desde un feed MRSS que publique. Video Cloud extrae los metadatos del feed de MRSS y crea un activo remoto o descargas (desde donde están alojados) y procesa los videos y cualquier archivo asociado (miniaturas, archivos de subtítulos cerrados, etc.).

Nota: Este tema se aplica a OIS v2.5.

Descripción general de la ingesta de MRSS

Sobre MRSS

Media RSS (MRSS) es una extensión de RSS que le permite sindicar archivos multimedia en fuentes RSS. Para obtener detalles técnicos, consulte el Especificación de medios RSS.

Configuración de MRSS

Comuníquese con su administrador de cuentas e indique que desea utilizar MRSS. Para cada video que desee ingerir, debe proporcionar a Ooyala URL de contenido válidas. Las URL pueden incluir http: // o https: //.

Usando MRSS

Para usar MRSS, usted aloja metadatos y / o videos y otros archivos (miniaturas, subtítulos) en un servidor web y crea un archivo de manifiesto XML que hace referencia a ese contenido.

Uso de MRSS para extraer metadatos únicamente

  1. Cree un archivo de manifiesto XML como se describe en Archivo de manifiesto XML. Debes usar el ooyala: activo remoto etiqueta para la ingestión de solo metadatos.
  2. Aloje el archivo de manifiesto XML en una de las ubicaciones de MRSS que proporcionó. Asegúrese de que el nombre del archivo y la ruta coincidan exactamente con una de las rutas que proporcionó a Ooyala.

Uso de MRSS para extraer videos, miniaturas y archivos de subtítulos ocultos

  1. Aloje uno o más archivos (videos, miniaturas y archivos de subtítulos) en un servidor web.
  2. Cree un archivo de manifiesto XML (consulte Archivo de manifiesto XML). Asegúrese de que las entradas del archivo hagan referencia a los archivos de su servidor. En particular, el valor de la <Contenido multimedia> elementos href atributo debe ser una URL HTTP para el archivo en su servidor, como en el siguiente ejemplo: 
    <media: content url = "http://mysite.com/upload/lacrosse_70.mov" />
  3. Aloje el archivo de manifiesto XML en una de las ubicaciones de MRSS que proporcionó. Asegúrese de que el nombre de archivo y la ruta coincidan exactamente con una de las rutas que proporcionó a Ooyala.

    Nota: Desafortunadamente, el concepto de "registro de clientes" no se aplica al procesamiento basado en fuentes de MRSS. La mejor forma de verificar que su feed se esté procesando correctamente es comprobar la apariencia de sus activos, ya sea con la interfaz de usuario de Video Cloud Studio o con el API de backlot /v2/activos solicitud.

Después de que se procesen los videos y cualquier otro archivo asociado, para extraer más archivos, simplemente aloje otro archivo MRSS en la ubicación acordada.
