Introducción

Utilizar el API de backlot para cargar contenido mediante programación en Video Cloud. Este enfoque le permite integrar su sistema de gestión de contenido (CMS) o flujos de trabajo directamente con la plataforma Video Cloud.

Tenga en cuenta que también puede migrar sus integraciones o aplicaciones a la API de CMS y API de ingesta dinámica en cualquier momento.

Puedes usar el API de backlot para cargar y administrar videos, miniaturas y archivos de subtítulos cerrados mediante programación.

Si desea ingerir contenido mediante la API de Backlot, consulte:

  • Activos de video y audio (para videos y miniaturas)
  • Trabajar con subtítulos ocultos

La llamada principal que se utiliza para cargar archivos (denominada "activos") es la /v2/activos ruta.

Nota: No agregue un video con un nulo external_id, es decir, un external_id sin valor"") o un valor de "nulo". Estos ID externos nulos no se pueden buscar más adelante.

Activos de video y audio

Cuando subes un vídeo a Ooyala, Ooyala lo transcodifica automáticamente utilizando la configuración especificada para tu cuenta y pone los vídeos disponibles para su reproducción a través del reproductor.

Tenga en cuenta que los activos de audio se almacenarán en Video Cloud como videos sin pista de video.

Le recomendamos que suba videos a Ooyala en H.264, pero también admitimos muchos códecs de uso común.

Para crear nuevos activos de video o audio:
  1. Crea el activo con propiedades básicas
  2. Recuperar las "URL de carga" del sistema.
  3. Para cada "URL de carga", coloque el fragmento de archivo correspondiente en el sistema
    Nota: Cuando cargue cada fragmento, no firme la solicitud, como lo haría con otras solicitudes. Simplemente COLOQUE el contenido del fragmento en las URL exactas. Estas URL ya se han firmado por ti.

Obtenga las URL de carga para los activos recién creados

    [GET] /v2/assets/{asset_id}/uploading_urls
Nota: Cuando cargue cada fragmento, no firme la solicitud, como lo haría con otras solicitudes. Simplemente COLOQUE el contenido del fragmento en las URL exactas. Estas URL ya se han firmado por ti.

Establecer estado de carga

    [PUT] /v2/assets/{asset_id}/upload_status { "status":"uploaded" }

Ver transmisiones de un video

El término arroyos en este contexto se refiere a los procesos internos de Ooyala que el sistema inicia para transcodificar sus videos.

Nota: No puede controlar ni manipular estos flujos; están controlados por el sistema. Solo puede ver sus nombres, no cambiarlos.
    [GET] /v2/assets/{asset_id}/streams

Ver archivos fuente para un video

    [GET] /v2/assets/{asset_id}/source_file_info

Obtener URL para imágenes de vista previa generadas automáticamente

    [GET] /v2/assets/{asset_id}/generated_preview_images

Cargar imagen de vista previa personalizada

    [POST] /v2/assets/{asset_id}/preview_image_files <file_contents>

Establecer la configuración de la imagen de vista previa principal

Establezca el tipo de imagen de vista previa principal de un activo en uno de los siguientes:
  • generado: use la imagen de vista previa generada automáticamente
  • uploaded_file: use la imagen de vista previa personalizada cargada
  • remote_url: URL de la imagen de vista previa
    [PUT] /v2/assets/{asset_id}/primary_preview_image { "type":"generated" | "uploaded_file" | "remote_url" }

Acerca de los metadatos

Un activo puede tener hasta 100 pares de nombre / valor de metadatos personalizados. Un nombre está limitado a 128 caracteres y su valor está limitado a 2048 caracteres.

Trabajar con subtítulos ocultos

Puede agregar subtítulos a videos o videos de activos remotos a través de la API de Backlot.

Formatos admitidos para subtítulos

Formatos admitidos
Extensión de archivo Descripción
.vtt (WebVTT /): este es el formato necesario para los reproductores de vídeo HTML5 WebVTT se puede utilizar para subtítulos integrados y secundarios en formatos de transmisión HTTP como HLS y DASH.
.TTML (antes DXFP) - se convertirá a WebVTT TTML (archivo de lenguaje de marcado de texto cronometrado. En general, un archivo de subtítulos opcionales de este tipo tendrá una extensión.dfxp (perfil de intercambio de formato de distribución).

Subir subtítulos ocultos

    [PUT] / v2 / assets /{asset_id}/subtítulos/{dfxp_file_content}

Obtener subtítulos ocultos

    [GET] / v2 / assets /{asset_id}/subtítulos

Actualizar subtítulos ocultos

    [PUT] / v2 / assets /{asset_id}/subtítulos

Eliminar subtítulos ocultos

    [BORRAR] / v2 / assets /{asset_id}/subtítulos
