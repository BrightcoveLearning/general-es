Uso de los reproductores Ooyala para entregar contenido de Video Cloud

Este tema describe cómo los reproductores web y SDK de Ooyala continuarán funcionando después de que sus videos se migren a Video Cloud, manteniendo sus videos en línea funcionando mientras completa la migración a los reproductores Brightcove.

Nota: este tema se aplica solamente a los clientes de Ooyala migrados a Video Cloud.

Introducción

Brightcove ofrece una reproductor web altamente personalizable , y también reproductores nativos para Android, iOS y tvOS.

Sin embargo, sabemos que reemplazar los códigos de inserción del reproductor y reconstruir las aplicaciones del dispositivo llevará tiempo, por lo que Brightcove ha creado un adaptador que permitirá que sus reproductores Ooyala existentes continúen trabajando recuperando los datos de video de Video Cloud en lugar de Backlot:

Tenga en cuenta que para probar el adaptador de reproducción, deberá trabajar con su CXM para realizar ajustes en la configuración de DNS en sus computadoras y dispositivos. Consulte Agregar el servidor DNS de Brightcove para obtener más información.

Cómo funciona

Todos los reproductores de Brightcove recuperan datos de listas de reproducción y vídeos a través del API de reproducción. Sin embargo, no es necesario configurar las solicitudes de API manualmente. El reproductor web tiene Catalogar biblioteca que maneja las solicitudes, y los SDK del dispositivo tienen métodos integrados para eso.

Para que sus reproductores Ooyala sigan funcionando, Brightcove ha creado un adaptador que apunta las solicitudes de los reproductores de datos de listas de reproducción y video a la API del reproductor y los mismos videos en Video Cloud.

Tenga en cuenta: ya no podrá modificar la apariencia de su reproductor web Ooyala. El aspecto y el tacto actuales se conservarán.

Analítica

En el futuro, utilizará Video Cloud Analytics. Las correcciones para permitir que sus reproductores de Ooyala existentes con sus videos de Video Cloud incluyen el envío de datos de análisis al sistema de análisis de Video Cloud. No es necesario realizar ninguna modificación en sus páginas o reproductores.

Puede acceder a los datos analíticos de Video Cloud utilizando los informes estándar y personalizados en el Módulo de analítica o el API de análisis. Ver Migración de la API de REST de Analytics a la API de Brightcove Analytics si obtendrá datos analíticos a través de la API.

Lo que puede no funcionar

Si tiene complementos integrados con el reproductor Ooyala, seguirán funcionando. Sin embargo, si estos complementos dependen de datos de video que no están disponibles, no funcionarán correctamente. Ver el mapeo de campos de listas de reproducción y video de Ooyala a campos de Video Cloud para más información.

Además, si tiene un complemento que realiza solicitudes a las API de Ooyala, estos no funcionarán en muchos casos, aunque algunos API de backlot los puntos finales seguirán estando operativos. La API Discovery no estará operativa.

Datos analíticos voluntad recopilarse y enviarse al sistema Brightcove Analytics.

Próximos pasos

Si bien Brightcove se compromete a mantener en funcionamiento los videos y las listas de reproducción de sus reproductores Ooyala durante la transición a Video Cloud, debe trabajar en un plan para pasar a los reproductores Brightcove a fin de aprovechar al máximo sus funciones y personalización.
