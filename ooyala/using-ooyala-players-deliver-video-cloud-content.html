    Uso de los reproductores de Ooyala para entregar contenido de Video Cloud

    En este tema se describe cómo los reproductores web y SDK de Ooyala seguirán funcionando después de migrar los vídeos a Video Cloud, lo que permite que los vídeos en línea funcionen mientras finaliza la migración a los reproductores de Brightcove.

    Nota: este tema sólo se aplica a los clientes de Ooyala migrados a Video Cloud.

    Introducción

    Brightcove ofrece un reproductor web altamente personalizable, así como reproductores nativos para Android, iOS y tvOS.

    Sin embargo, sabemos que la sustitución de los códigos de inserción del reproductor y la reconstrucción de las aplicaciones del dispositivo llevará tiempo, por lo que Brightcove ha creado un adaptador que permitirá a los reproductores Ooyala existentes seguir trabajando recuperando los datos de vídeo de Video Cloud en lugar de Backlot:

    Tenga en cuenta que para probar el adaptador de reproducción, deberá trabajar con su CXM para realizar ajustes en la configuración de DNS en sus equipos y dispositivos. Consulte Agregar el servidor DNS de Brightcove para obtener más información.

    ¿Cómo funciona?

    Todos los reproductores de Brightcove recuperan datos de vídeo y listas de reproducción a través de la API de reproducción. Sin embargo, no tiene que configurar las solicitudes de API manualmente. El reproductor web tiene una biblioteca de catálogo que gestiona las solicitudes, y los SDK del dispositivo tienen métodos integrados para ello.

    Para que sus reproductores Ooyala funcionen, Brightcove ha creado un adaptador que señala las solicitudes de los reproductores de datos de vídeo y listas de reproducción a la Player API y a los mismos vídeos en Video Cloud.

    Ten en cuenta que ya no podrás modificar el aspecto de tu reproductor web Ooyala. El aspecto y la sensación actuales se mantendrán.

    Analytics

    En el futuro, utilizará Video Cloud Analytics. Las correcciones para permitir a los reproductores Ooyala existentes con sus vídeos de Video Cloud incluyen el envío de datos analíticos al sistema de análisis de Video Cloud. No hay ninguna modificación que tengas que hacer en tus páginas o jugadores.

    Puede acceder a los datos de análisis de Video Cloud utilizando los informes estándar y personalizados del módulo Analytics o la API de Analytics. Consulte Migración de la API REST de Analytics a la API de Brightcove Analytics si va a obtener datos de análisis a través de la API.

    Lo que puede no funcionar

    Si tienes complementos integrados con el reproductor Ooyala, seguirán funcionando. Sin embargo, si estos complementos dependen de datos de vídeo que no están disponibles, no funcionarán correctamente. Consulte la asignación de campos de vídeo y lista de reproducción de Ooyala a campos de Video Cloud para obtener más información.

    Además, si tiene un complemento que realiza solicitudes a las API de Ooyala, estas no funcionarán en muchos casos, aunque algunos Backlot API endpoints seguirán funcionando. La API de detección no estará operativa.

    Los datos de análisis se recopilarán y enviarán al sistema Brightcove Analytics.

    Siguiente pasos

    Aunque Brightcove se compromete a mantener los vídeos y listas de reproducción en tus reproductores de Ooyala trabajando en la transición a Video Cloud, deberías estar trabajando en un plan para pasar a los jugadores de Brightcove con el fin de aprovechar al máximo sus características y personalizabilidad.
